## 1. THE "VIBE-CODED" LOOK ‚Äî WHAT WE'RE FIGHTING

Based on extensive industry research [1][2][3], these are the exact visual
crutches that make users instantly recognize an AI-generated app:

| "AI Starter Pack" Default         | Why It's Detected                              | Our Override                                  |
| --------------------------------- | ---------------------------------------------- | --------------------------------------------- |
| Purple-to-blue gradient           | Every model's favorite gradient                | Flat slate palette with hard 1px borders       |
| Inter / Roboto / system sans      | Default font in every starter template         | `JetBrains Mono` (data), `Space Grotesk` (UI) |
| `rounded-lg` (8px)                | Mathematical average of every SaaS card        | `rounded-none` (brutalist) or `rounded-full`   |
| Lucide icons                      | Default icon pack in Shadcn/v0                 | Phosphor or inline SVG custom icons            |
| Glassmorphism with `blur-md`      | Lazy 12px blur that looks washed out           | Liquid Glass 2.0: 25px blur, 150% saturate     |
| "Unleash your potential! üöÄ"      | Robotic marketing speak + emojis               | Journalistic, data-driven, zero emojis         |
| Excessive whitespace w/o purpose  | Masks lack of content strategy                 | Dense, information-rich grid layouts            |
| Generic "No data found" empties   | Zero design effort on edge cases               | Contextual empty states with market intelligence|

## 2. THE SIX PROTOCOLS

### Protocol 1: Negative Prompting (BAN THE DEFAULTS)

Every AI prompt that touches UI must include explicit bans. These go in
`.cursorrules`, system prompts, or `KNOWLEDGE_UI_STANDARDS.md` (this file).

```
BANNED:
  colors:     [purple, cyan, electric blue, #7C3AED, #06B6D4, #3B82F6]
  gradients:  [linear-gradient with purple/blue, "aurora" effects]
  fonts:      [Inter, Roboto, system-ui as display font]
  radii:      [rounded-md, rounded-lg ‚Äî the 6-8px generic zone]
  icons:      [emoji in UI text or code comments]
  copy:       ["Unleash", "Dive in", "Get started", "Your journey"]
  components: [unmodified Shadcn defaults, generic hero sections]
```

**Why this matters:** AI models take the path of least resistance [1]. If you
don't explicitly ban these, they will appear in every generated component,
creating a homogenized "SaaS template" look that erodes brand trust.

### Protocol 2: Visual Anchoring (STEAL LIKE AN ARTIST)

Text-only prompts produce generic results. Visual anchoring produces
specific, intentional designs [4].

**Workflow:**
1. Screenshot a reference UI from **Mobbin**, **Godly.website**, or **Dribbble**
2. Feed it to the AI with: *"Extract the typography hierarchy, spacing grid,
   and structural layout. Apply it to [specific component]."*
3. Never say "make it beautiful" ‚Äî always say "match this exact grid density"

**Reference Design Systems for SportSync:**
- BetMGM dark theme (2025 redesign) ‚Äî dense data, dark backgrounds [5]
- DraftKings prop tracking cards ‚Äî sharp geometry, live data emphasis [6]
- Bloomberg Terminal ‚Äî brutalist data density, monospace dominance

### Protocol 3: BYODS (Bring Your Own Design System)

Your `index.css` already has a strong foundation. Here's the canonical
token set that must NEVER be overridden by inline styles:

```css
/* CANONICAL TOKENS ‚Äî index.css ¬ß1 */
:root {
  --color-background: #F8FAFC;   /* slate-50 */
  --color-surface:    #FFFFFF;
  --color-text-primary: #0F172A; /* slate-900 */
  --color-text-secondary: #64748B; /* slate-500 */
  --color-border:     #E2E8F0;   /* slate-200 */
  --ease-spring:      cubic-bezier(0.2, 0.8, 0.2, 1);
  --ease-apple:       cubic-bezier(0.25, 0.1, 0.25, 1.0);
}
```

**Rules:**
- ALL colors must reference these variables or Tailwind's slate palette
- NO hex codes invented inline (e.g., `text-[#a855f7]` is banned)
- Spacing uses Tailwind's 4px base scale exclusively: `p-1` through `p-8`
- The monospace data font is `JetBrains Mono` (loaded in index.css)

### Protocol 4: Swap the AI Starter Pack

| Default              | Our Override                         | Why                                    |
| -------------------- | ------------------------------------ | -------------------------------------- |
| Inter (body)         | System `-apple-system` stack         | Native feel, zero FOUT                 |
| Inter (display)      | `JetBrains Mono` or `Space Grotesk` | Technical, data-driven personality     |
| Lucide icons         | Phosphor (thin weight) + inline SVG  | Sharper, thinner, more distinctive     |
| Shadcn default theme | Custom tokens in index.css           | Prevents "Vercel-chic" generic look    |
| `rounded-lg` (8px)   | `rounded-none` / `rounded-sm` (2px)  | Brutalist editorial feel               |
| `bg-purple-500`      | `bg-slate-900` / `bg-white`          | Monochromatic, institutional           |

### Protocol 5: Human Polish (THE 20% RULE)

After AI generates any component, manually modify at least 20% of it [1][7]:

**Micro-interactions (mandatory on ALL interactive elements):**
```tsx
// GOOD: Tactile spring with subtle scale
<motion.div
  whileHover={{ scale: 1.02, y: -2 }}
  whileTap={{ scale: 0.98 }}
  transition={{ type: "spring", stiffness: 400, damping: 25 }}
>

// BAD: No interaction at all, or just opacity fade
<div className="hover:opacity-80">
```

**Empty States (mandatory ‚Äî never "No data found"):**
```tsx
// GOOD: Contextual intelligence
<div>Market lines locked. Awaiting tip-off at 7:00 PM ET.</div>

// BAD: Generic placeholder
<div>No data found.</div>
```

**Copy Rules:**
- Headlines: ALL CAPS with colons (`MATCHUP CONTEXT:`)
- Labels: `text-[10px] uppercase tracking-[0.15em] font-semibold text-slate-400`
- Data: `font-mono tabular-nums` with `JetBrains Mono`
- Never use exclamation marks, emojis, or marketing adjectives

### Protocol 6: Component-by-Component Iteration

**Never** prompt "make the whole app look beautiful." This overwhelms the
context window and produces CSS soup [8].

**Instead, iterate surgically:**
```
Step 1: "Fix the nav bar. Text should be 11px uppercase tracking-[0.15em].
         Add a 1px solid bottom border using var(--color-border)."

Step 2: "The card padding is wrong. Use px-5 py-4. Remove the rounded-lg
         and use rounded-none. Add a top-left accent stripe."

Step 3: "The empty state says 'No data found.' Replace with 'Market lines
         locked. Awaiting tip-off.' Center it vertically."
```
